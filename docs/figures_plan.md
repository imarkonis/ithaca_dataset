# Figure Plan

## Purpose

This document defines the planned figures for the standalone ITHACA dataset paper.
It aligns the team on: the role of each figure, its main message, panel structure,
and required scripts/outputs.

## Scoping principle

ESSD assesses the **significance, quality, and reusability of the data**, not the
scientific interpretation of the change signal. Interpretation belongs to the
companion analysis paper. Every figure below must answer one of:

* *What is in this dataset?* (structure, coverage, provenance)
* *How good is it?* (validation, agreement, convergence)
* *How do I use it, and where should I not trust it?* (masks, sensitivity, records)

Figures whose main message is a scientific claim about intensification or drying
belong in the analysis paper, not here. A single demonstration figure showing what
the product *enables* is appropriate; a results narrative is not.

# Figure 1 — Conceptual methodology diagram

## Working title

**Methodological framework for the probabilistic TWC dataset**

## Main purpose

Explain the workflow from input datasets to the released products.

## Main message

The dataset is not a single merged product. It is a **probabilistic ensemble of
plausible P–E worlds**, generated by evaluating datasets, assigning region × biome
selection probabilities, and sampling them by Monte Carlo.

## Suggested content

Schematic flow with these blocks:

1. **Input datasets** — five coherent P–E analysis pairs; two 8-member reference
   ensembles (evaluation only); PET products; auxiliary validation data
   (GRACE, CCI, ROBIN) as a **side input into the validation block**, not the pipeline.
2. **Harmonisation** — common 0.25° annual grid, 1982–2021, `twc_complete_grid` mask.
3. **Evaluation** — leave-one-out reference; climatology, variability, trend,
   significance agreement; physical plausibility screening.
4. **Weighting** — grid-cell performance → region × biome probabilities.
   Show the **base** scenario in the main path; show the other seven as a labelled
   branch feeding Figure 6 (they are distributed records, not a side note).
5. **Monte Carlo ensemble** — one dataset sampled per region × biome per member.
6. **Derived products** — P, E, P−E (availability), (P+E)/2 (flux), storylines,
   at grid / region × biome / region / global scales.

## Suggested design

* Clean vertical workflow, 6 stages, colour by stage only.
* Mark which stage each released data record exits from — this doubles as a
  provenance map and lets Table T3 reference it.
* Script/file details go in the caption, not the figure.

## Inputs

Schematic. No script.

# Figure 2 — Anatomy of a sampling unit (worked example)

## Working title

**From region × biome units to a probabilistic ensemble: the Mediterranean example**

## Main purpose

Walk the reader through the full construction on a **single, legible region**, so the
machinery of Figure 1 becomes concrete before any global product is shown.

## Main message

Within one IPCC region, each biome is an independent sampling unit with its own
dataset probabilities; drawing one dataset per unit per member yields an ensemble
whose spread *is* the dataset-choice uncertainty.

## Why this is main text

A global weight map shows the *result* of the method; this shows the *method*. It
makes the region × biome atom, the base weights, and the Monte Carlo spread reviewable
in one glance, using a region a reader can name. `07a` already spot-checks MED in its
validation block, so the example is natural.

## Suggested content

3-panel, base scenario, one region (**MED** suggested):

* **(a)** Region and biome unit map (categorical) for the chosen region — each biome
  within MED as a distinct sampling unit, on the 0.25° grid.
* **(b)** Dataset weights per unit: **stacked barplot**, one bar per biome, bars
  summing to 1, segments coloured by dataset. Base scenario.
* **(c)** Time series of the regional mean from the probabilistic ensemble: MC members
  as thin transparent grey lines, ensemble median in black.

## Design notes

* Panels read left → right as *structure → probability → consequence*. Keep dataset
  colours identical between (b) and any per-member colouring in (c).
* (b) sums to 1 per bar by construction — `07a` asserts this
  (`abs(sum(w_region_biome) − 1) < 1e-6`) — so state in the caption that the bars are
  exact, not normalised for display.
* (c) is the right place to show the ensemble *honestly*: grey spaghetti reveals that
  members are discrete recombinations of five worlds, whereas a smooth 5–95 % band at
  this scale would imply a continuum that does not exist (see Figure 5). **Do not
  replace the spaghetti with a band.**
* Consider annotating (c) with the number of members drawn (500, base) and the number
  of distinct unit-combinations available in this region.

## Which variable in (c)?

Recommend **availability (P − E)**, matching the paper's headline quantity, with P and
E versions in **S6**. Alternatively a two-row (c) with P and E stacked.

## Inputs

* `01g_set_region_classes.R` → `twc_grid_classes.Rds` (region, biome per cell)
* `03f_estimate_regional_weights.R` → `weights_region_biome.Rds` (base weights)
* `04b_create_aggregated_mc_ensemble.R` → regional annual ensemble (members)
* `04d_estimate_ensemble_change.R` → ensemble median

## Open decisions

* **Which region?** MED is legible, well-sampled, and already spot-checked in `07a`.
  A region with more biome diversity may demonstrate the unit structure better —
  MED may yield too few bars in (b). Worth one comparison before fixing.
* Panel (c): availability only, or P and E as stacked sub-panels?
* Overlay the five coherent dataset worlds in (c) as reference lines, or keep it to
  members + median?

# Figure 3 — Dataset coverage after the physical-plausibility gates

## Working title

**Effective spatial coverage of each candidate dataset after plausibility screening**

## Main purpose

Show, per dataset, where the product exists — and by extension where it does not.

## Main message

The plausibility gates remove cells unevenly across datasets. Coverage is not a
property of the grid; it is a property of each dataset on that grid.

## Why this is main text

This is the single most important figure for a reusability assessment. A user
downloading the product needs the retained mask before anything else, and needs to
know that the surviving candidate set differs by location. The `n_below_pet > 6` and
aridity gates are the main source of high-latitude holes.

## Suggested content

**5 maps, one per analysis dataset** (ERA5L, FLDAS, GLEAM, MERRA, TERRA), each showing
retained vs excluded cells after both gates.

Preferred encoding — distinguish excluded cells by *reason*, not just presence:

* retained
* excluded, aridity check (P/E < 0.9)
* excluded, PET consistency (`n_below_pet ≤ 6`)
* excluded, both

This turns a binary mask into a diagnostic. Since `n_below_pet` is a per-cell property
of the PET products (identical across the five datasets) while the aridity check is
per-dataset, the panels will share their PET-driven holes and differ in their
aridity-driven ones — which is itself the figure's point and should be said in the caption.

Annotate each panel with **percent land coverage retained**, so the five are comparable
at a glance.

## Suggested design

* Consistent projection and grey for excluded cells across all five.
* Same grey reused in Figure 4 for gate-excluded cells, so the reader links hole to cause.
* Small multiples, one row of five, or 2×3 with a shared legend.

## Inputs

* `03c_dataset_aridity_check.R` → per-dataset aridity flag
* `03d_pet_check.R` → `n_below_pet` per cell
* `03e_dataset_agreement_weights.R` → the gate as actually applied before weighting
* `01g_set_region_classes.R` → land mask, region outlines

## Open decisions

* Four-class encoding (by reason) or binary retained/excluded? Four-class recommended.
* Overlay IPCC region outlines, or keep the maps clean?
* Report retained coverage as % of land cells, or % of land area (area-weighted)?
  Area-weighted is more honest given the latitudinal concentration of the holes.

# Figure 4 — Dataset weights and dataset-choice uncertainty

## Working title

**Dataset selection probabilities and where dataset choice is uncertain**

## Main purpose

Show which datasets dominate spatially, and where the five candidate worlds are close
to equiprobable.

## Main message

Some regions are consistently represented by one dataset; elsewhere several datasets
are near-equally plausible, and that equipoise is itself a product.

## Suggested content

4-panel, base scenario:

* **(a)** Dominant precipitation dataset (categorical, argmax weight)
* **(b)** Dominant evaporation dataset (categorical)
* **(c)** Effective number of datasets, precipitation (continuous)
* **(d)** Effective number of datasets, evaporation (continuous)

## Uncertainty metric — decided

Use the **effective number of datasets** (inverse-Simpson, `1 / Σwᵢ²`), already computed
in `03e`. It is bounded on **[1, 5]** with an immediate reading:

* → 1 : one dataset owns this cell (low dataset-choice uncertainty)
* → 5 : full equipoise among the five worlds (high uncertainty)

Comparable across scenarios and across P and E. Weight range has neither a natural
bound nor an interpretation; top-minus-second discards the tail.

## Dropped from the earlier plan

**"Least representative dataset" is cut.** In most cells it is the argmin of a near-flat
weight vector, so it is spatially incoherent noise; where it *is* coherent it
approximates the `inverted` scenario's dominant-dataset map. Move to **S4** if wanted.

## Suggested design

* Fixed dataset colours (see consistency rules) for (a)–(b).
* Single sequential ramp for (c)–(d), shared scale.
* Gate-excluded cells in the same grey as Figure 3.

## Inputs

* `03e_dataset_agreement_weights.R` → grid weights, inverse-Simpson
* `03f_estimate_regional_weights.R` → `weights_region_biome.Rds`

## Open decisions

* Base scenario only (recommended), with the cross-scenario view in Figure 6 / S4.
* Grid-level or region × biome-level weights? Grid shows the field; region × biome shows
  what is *actually sampled*. Possibly (a)–(b) at unit level, (c)–(d) at grid.
* Given Figure 6 also maps dominant datasets, is (a)–(b) redundant? If so, Figure 4
  reduces to the two inverse-Simpson panels and Figure 6 carries all dominance.

# Figure 5 — Change in precipitation and evaporation

## Working title

**Global precipitation and evaporation change across plausible dataset worlds**

## Main purpose

Show what the product enables — the spread of plausible ΔP and ΔE, and where that
change is located.

## Main message

The probabilistic dataset resolves the spread of plausible change rather than
asserting a single answer. Where the ensemble concentrates, the signal is robust to
dataset choice; where it spans quadrants, it is not.

## Coordinate system

**ΔP–ΔE space**: x = ΔP, y = ΔE. Water flux is no longer plotted (scope decision 1).

Read the plane correctly and label it as such:

* **Quadrants are driver signs**: ↑P↑E, ↓P↑E, ↑P↓E, ↓P↓E.
* **The 1:1 diagonal is Δ(P−E) = 0** — it separates wetting (below, ΔP > ΔE) from
  drying (above, ΔE > ΔP). Availability is the *perpendicular distance* from this line.
* Distance from the origin is the magnitude of change.

Do **not** additionally shade "storyline quadrants": in this rotation the storylines
are diagonal wedges, not quadrants. Draw the 1:1 line and label wetting/drying on
either side of it.

## Suggested content

* **(a)** Global ΔP–ΔE scatter. MC ensemble members as small semi-transparent points;
  ensemble median; the five coherent dataset worlds as large labelled points; the
  unweighted dataset mean as an open diamond. 1:1 line, quadrant labels, optional
  density contour for the member cloud.
* **(b)** Global map of ΔP (ensemble median).
* **(c)** Global map of ΔE (ensemble median).

Maps (b)–(c) share a diverging colour scale centred on zero so they are directly
comparable. A Δ(P−E) map is optional and may be redundant given (a) reads availability
off the diagonal.

## Caveat that must be stated

**`04c` is empty.** The gridded MC record is produced by region × biome → grid
extrapolation in `04d`, so every cell inside a unit carries the *same* value: maps
(b)–(c) will be **piecewise-constant over sampling units**, not smooth fields. The
deterministic scenario means from `07a` *do* vary within units. These are different
products and must not be mixed in one figure. Either:

* map the `04d` extrapolated ensemble median and show the unit polygons honestly; or
* map the `07a` base scenario mean and label it a deterministic scenario mean.

The caption must say which. See "Blocked" below.

## Uncertainty display — caution

Within one region × biome unit, MC sampling draws **one of five** datasets, so the
per-unit distribution has at most five atoms. A "5–95 % band" at unit or regional level
is a quantile of a five-point discrete mixture: it will land exactly on individual
dataset values and jump discontinuously. Continuous bands are defensible for **global**
aggregates (many units average) and misleading below that.

## Inputs

* `04b_create_aggregated_mc_ensemble.R` → aggregated global ensembles (panel a)
* `04d_estimate_ensemble_change.R` → change metrics, gridded extrapolation (b, c)
* `07a_estimate_scenario_means.R` → deterministic scenario-mean grid (alternative b, c)
* `07b_…_global.R` → `global_avail_flux_change_plot_dt.Rds` — **requires rotation to
  ΔP–ΔE**; `prec_change` and `evap_change` are already columns in this table, so the
  data exists and only the plotting layer changes.

## Open decisions

* Panels (b)–(c): `04d` extrapolation or `07a` scenario mean? (See caveat.)
* Absolute change only, or an inset for relative change?
* Include a Δ(P−E) map as (d), or leave availability to the diagonal in (a)?

# Figure 6 — Dominant-dataset flip across weighting scenarios

## Working title

**Where the dominant dataset depends on the weighting design**

## Main purpose

Show, in one map, how much the weighting choice changes which dataset the product
actually selects.

## Main message

Over most of the land surface the dominant dataset is stable across defensible
weighting choices; where it flips, dataset-choice uncertainty and weighting design
interact and the product should be used with care.

## Suggested content

A **flip map**: per cell, a summary of how the argmax dataset varies across scenarios.

Candidate encodings, in order of preference:

1. **Number of distinct dominant datasets** across the four *plausible* scenarios
   (`base`, `clim_dominant`, `evap_dominant`, `prec_dominant`). Bounded 1–4, immediate
   reading: 1 = stable, 4 = the weighting decides the answer.
2. **Binary flip vs base**: cells where any plausible scenario disagrees with `base`.
   Simpler, less informative.
3. Dominant dataset under `base`, with flip cells hatched. Combines Figure 4a–b with
   this figure, and may let Figure 4 lose its dominance panels entirely.

Recommend **encoding 1**, possibly as two panels (precipitation, evaporation).

## Which scenarios enter the flip?

**Only the four plausible scenarios.** `neutral` is the 1/n null and `inverted` is an
adversarial mirror — including them inflates the flip count with cells where nothing
scientifically meaningful disagrees. `rank_linear`/`rank_exp` are transform foils and
may be shown as a separate flip count, or held for S4.

State this in the caption. The eight scenarios are **not interchangeable**:

* **Plausible worlds** (`base`, `clim_dominant`, `evap_dominant`, `prec_dominant`) —
  their spread *is* the methodological uncertainty.
* **`neutral`** — exact 1/n null reference; distance from it measures how much the
  weighting does at all.
* **`inverted`** — adversarial mirror, a lower bound on skill. A diagnostic, not a world.
* **`rank_linear`, `rank_exp`** — transform foils; `rank_exp` is near winner-take-all.

## This is not only a robustness check

All eight weight fields are **distributed as data records**. The scenario family is
therefore a description of the product, not merely a supporting analysis.

## Inputs

* `03e_dataset_agreement_weights.R` → per-scenario grid weights (argmax per scenario)
* `03f_estimate_regional_weights.R` → `weights_region_biome.Rds` (if mapped at unit level)

## Open decisions

* Flip over grid cells or over region × biome units? Units are what is sampled, so a
  unit-level flip map is arguably the truer statement.
* One panel or two (P and E)?
* Does this absorb Figure 4a–b? (See Figure 4 open decisions.)

# Cross-figure consistency rules

## Dataset colours

Fixed across all figures, as set in `07b`:

```
ERA5L  #1b9e77   FLDAS  #d95f02   GLEAM  #7570b3
MERRA  #e7298a   TERRA  #66a61e
```

## Scenario colours and labels

As set in `07b`. Plausible scenarios filled; `neutral` and `inverted` open.
Labels: Base, Climate-dominated, Evaporation-dominated, Precipitation-dominated,
Rank-linear, Rank-exponential, Neutral, Disagreement.

## Notation

* **P** precipitation, **E** evaporation
* **P − E** water availability

Water flux `(P+E)/2` is **not plotted** in this paper. It remains a derived product in
the pipeline (Figure 1, block 6); define it once in the Data Records section and
nowhere else. Storylines likewise: a data record, not a figure.

## Period

Always state: **1982–2001** and **2002–2021**; change = late minus early.

## Ensemble summaries

* Global aggregates: **median**, with 5–95 % band.
* Region × biome and regional: **no continuous band** — show member spaghetti or the
  selection probabilities over the five datasets (the distribution has ≤ 5 atoms).
* Gate-excluded cells: rendered in a single identical grey in every map.

## ΔP–ΔE panels

Quadrants = driver signs. The 1:1 diagonal = Δ(P−E) = 0, separating wetting from
drying. Never shade quadrants as storylines in this space.
